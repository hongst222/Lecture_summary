Javascript 내장 기능

내장 객체 (API)

개발자가 호출할 수 있는 or 활용할 수 있는 기본 기능의 모음

MDN에서 많은 API에 대해 알 수 있음.

isNaN (Not a Number ⇒ 숫자가 아님.)

숫자가로 변환이 되는가 안되는가를 판단하기 위함.

  • 숫자면 false 숫자가 아니면 true
  • 숫자로 변환 가능한 형식일 경우 false;

Parse

parseFloat()

  • 주어진 값에서 변환한 부동소수점 수(실수)를 리턴
    • 변환이 불가하면 NaN을 리턴
    • ex) parseFloat(314e-2) = 3.14

parseInt(value, int)

  • value를 int진법 수를 기준으로 10진수 값으로 변환한다.
    • 변환이 불가하면 NaN을 반환
    • 숫자 + 글자 형태의 문자열은 숫자 부분만 취함
    • 숫자 + 글자 + 숫자 인 경우 “글자+숫자”로 문자열이 되므로 맨앞에 있는 숫자만 변환
    • 글자 + 숫자 형태의 문자열은 변환 불가 - NaN
    • 소수점을 포함할 경우 정수부분만 취한다.

encode, decode

encodeURI(string)

  • 주어진 문자열을 URL에 포함시키기에 적절한 형태로 변환(=인코딩)하는 처리
    • 인코딩하지 않는 문자
      • A-Z a-z 0-9 ; , / ? : @ & = + $ - _ . ! ~ * ( ) ‘ #

decodeURI(string)

코딩 된 문자열을 원래의 문자열로 역변환 (디코딩)

encodeURIComponent(string)

  • 알파벳과 숫자 및 비예약 표식을 제외한 모든 글자를 URL에 포함시킬 수 있는 문자열로 인코딩한다.
    • ⇒ URL에 사용해도 문제가 없는 특수문자를 제외하고 모든 글자를 변환

decodeURIComponent(string)

  • 인코딩 된 문자열을 원래의 문자열로 역변환 (디코딩)

setTimeout(func,int)

  • 지정된 함수를 두 번째 인자로 전달된 시간 후에 실행하도록 예약한다. ( 딜레이 가능)
    • setTimeout() 이후의 처리 로직들은 func의 실행 여부와 상관없이 즉시 실행된다.

setInterval(func, int)

  • 지정된 함수를 두 번째 인자로 전달된 시간마다 한 번씩 호출한다. ( 타이머기능)
  • setInterval() 이후의 처리 로직들은 func의 실행 여부와 상관없이 즉시 실행된다.
  • 타이머를 종료시킬 수 있는 timerid를 반환한다.
  • 이 값을 clearInterval() 함수에 전달하면 타이머 종료.
  • int는 1/1000초 (ms)를 의미하는 함수
    • 멀티태스킹에 이용된다.

String 메소드

[문자열 처리]

문자열은 그 자체가 객체.

객체라는 것은 그 안에 멤버변수(프로퍼티)와 메서드(함수)가 내장되어 있음을 의미

⇒ 일반적으로 생성하는 문자열 변수 안에는 메서드와 프로퍼티가 자동 내장

ex)

const a = “hi world”;

a.메서드()

문자열 객체에 내장된 메서드들은 변수가 담고 있는 내용을 가공하거나 특정 내용을 추출하는 기능을 갖는다. 이 기능들의 공통점은 원본 내용은 절대 변하지않고, 가공 결과를 리턴한다.

length

문자열의 글자 수를 가져온다 (공백 및 특수문자 포함)

charAt(value)

파라미터로 설정된 위치(value)의 글자를 리턴한다

indexOf(”str“)

파라미터로 전달된 내용(str)이 처음 나타나는 위치를 리턴한다.

  • 찾지 못할 경우 -1을 반환한다.

IndexOf(”str”,value)

value 위치에서 부터 파라미터로 전달된 내용(str)을 처음 찾는 위치를 리턴한다.

  • 찾지 못할 경우 -1 반환

lastIndexOf(”str”)

파라미터로 전달된 내용(str)이 마지막으로 나타나는 위치를 리턴한다.

  • 문자열의 처음부터 센다.
  • 찾지 못할경우 -1 반환
  • 파일 확장자를 찾을 때 유용하게 사용된다.

substring(value1,value2)

잘라내기 위한 시작위치(value1)와 끝 위치(value2)를 파라미터로 설정한다.

  • 지정된 끝위치 직전 글자까지만 잘라낸다.
  • 두 번째 파라미터가 없을 경우 지정된 위치부터 끝까지 자른다.

toUpperCase()

모든 글자를 대문자로 변환한다.

toLowerCase()

모든 글자를 소문자로 변환한다.

trim()

문자열의 앞/뒤 공백을 지운다.

split(”구분자”)

문자열에 포함된 구분자를 기준으로 문자열을 잘라 배열로 반환한다.

replace(”str”, “str”)

첫 번째 파라미터의 내용을 두번째 파라미터로 변경한 결과를 반환한다.

  • 첫 번째 파라미터와 일치하는 내용이 둘 이상 존재할 경우 첫번째 항목만 변경한다.

replaceAll(”str1”, “str2”)

첫 번째 파라미터의 내용을 두번째 파라미터로 변경한 결과를 반환한다.

  • 모든 경우를 변경한다.

숫자 처리와 관련된 기본 기능들을 제공하는 내장 클래스.

객체 생성

let a = new Number(123);

⇒ 이런식으로 생성할 경우 a는 객체가 된다.

Number()함수를 통한 생성

let a = Number(’123’);

⇒ a는 숫자형이 된다.

(정적)속성

표현 가능한 가장 작은 양수

Number.NaN

숫자가 아님을 나타내는 특별한 값

⇒ ex) isNaN(Number.NaN) ⇒ true를 return

(정적) 메소드

  • ex)

    • 변환이 불가하면 NaN을 반환
    • 숫자 + 글자 형태의 문자열은 숫자 부분만 취함
    • 숫자 + 글자 + 숫자 인 경우 “글자+숫자”로 문자열이 되므로 맨앞에 있는 숫자만 변환
    • 글자 + 숫자 형태의 문자열은 변환 불가 - NaN
    • 소수점을 포함할 경우 정수부분만 취한다.

Array 메소드

Array.push(v1,v2…,vn)

배열의 맨 끝에 요소를 추가한다. (파라미터 수 제한 없음)

Array.pop();

배열의 마지막 요소를 리턴하고 제거한다.

Array.shift();

배열의 맨 앞 요소를 리턴하고 제거한다.

Array.unshift(v1,v2,…vn)

배열의 맨 앞에 요소를 추가한다. (파라미터 수 제한 없음)

Array.splice(index,num)

index번째 위치부터 num개를 잘라서 반환하고 원본에서 제거한다.

Array.splice(index,num, v1,v2,…vn)

index번째 위치부터 num개를 잘라서 반환하고 그 위치에 v1~vn을 배치한다.

  • 제거한 수 보다 추가되는 원소 개수가 많을 경우 배열이 확장된다.
  • 기존의 원소들은 뒤로 밀린다.
  • 제거한 수 보다 추가되는 원소 개수가 적을 경우 배열이 축소 된다.

삭제할 원소의 수(num)을 0으로 지정하면 중간 삽입효과가 있다.

⇒ 기존의 원소들은 뒤로 밀려난다

Array.concat(arr1,arr2)

Array에 arr1과 arr2를 결합한 새로운 배열을 반환한다.

Array.indexOf(item,from)

인덱스 from부터 시작해 item(요소)를 찾는다.

  • 요소를 발견하면 해당 요소의 인덱스를 반환한다.
  • 두번째 파라미터(from)이 없으면 처음부터 탐색한다.
  • item(요소)를 발견하지 못하면 -1를 반환한다.
  • false를 검색하면 정확히 false만을 검색할뿐, 0을 검색하지는 않는다.
  • 요소를 찾을때 완전 항등 연산자 “===” 를 사용한다

    ⇒ NaN을 제대로 처리하지 못한다. (완전 항등 비교는 NaN에는 동작하지 않기 때문이다.)

Array.lastIndexOf(item,from)

인덱스 from부터 시작해 item(요소)를 찾는다.

  • 요소를 발견하면 해당 요소의 인덱스를 반환한다.
  • 두번째 파라미터(from)이 없으면 끝부터 탐색한다.
  • item(요소)를 발견하지 못하면 -1를 반환한다.
  • false를 검색하면 정확히 false만을 검색할뿐, 0을 검색하지는 않는다.
  • 요소를 찾을때 완전 항등 연산자 “===” 를 사용한다

    ⇒ NaN을 제대로 처리하지 못한다. (완전 항등 비교는 NaN에는 동작하지 않기 때문이다.)

Array.includes(item,from)

인덱스 from부터 시작해 item이 유무를 확인한다.

  • 해당하는 요소를 발견하면 true를 반환한다.
  • 발견하지 못한다면 false를 반환한다.
  • false를 검색하면 정확히 false만을 검색할뿐, 0을 검색하지는 않는다.
  • NaN도 제대로 처리한다.

배열을 탐색하는 방법

Array.forEach((element,index) ⇒ { 실행코드 })

배열의 모든 원소를 탐색한다.

  • 콜백 함수에게 배열의 값(element)과 인덱스(index)를 전달한다.
  • 콜백 함수는 원소의 수 만큼 순차적으로 실행된다.
  • forEach의 콜백함수에서 반복을 중단하고자 return을 사용할 경우 현재 동작중인 콜백만 종료될 뿐 전체 반복에는 영향이 없다.
  • break; 사용 불가 (break는 for,while문에서만 가능, 함수 안에서는 불가능)

    ex)

    code

    const arr = [10,20,30,40];
arr.forEach((element, index) => {
		console.log("%d번째 원소 --> %d, index, element);
});

    출력

    0번째 원소 --> 10
1번째 원소 --> 20
2번째 원소 --> 30
3번째 원소 --> 40

배열의 원소를 가공하여 새로운 배열 만들기

Array.map((element,index) ⇒ {실행코드})

map함수에 전달된 콜백이 리턴하는 값들을 하나의 배열로 묶어서 다른 배열(A)에 저장.

다른 배열(A)은 반드시 원본배열(Array)와 같은 길이를 갖는 배열이여야 한다.

return하지 않은 index에 대한 원소는 undefined가 된다.

ex)

code

const arr4 = [10, 20, 30, 40, 50];
const hello = arr4.map((element, index) => {
    return element * 10;
});
console.log(hello);

출력

[ 100, 200, 300, 400, 500 ]

화살표 함수(=>)는 처리로직이 한줄인 경우 “{ }” 와 “ ; “ , “return” 키워드가 생략 가능

ex)

code

const arr5 = [10,20,30,40,50];
const world = arr5.map((element, index) => element*10);
console.log(world);

출력

[ 100, 200, 300, 400, 500 ]

배열 검색

Array.find( function (element) { 함수내용 })

주어진 판별함수를 만족하는 첫번째 값을 반환한다. 못찾으면 undefined를 반환한다.

찾고자 하는 항목이 아닌 검색 규칙을 구현한 콜백함수를 전달해야 한다.

ex)

code

const arr6 = [5,12,8,131,44];
const found = arr6.find(function(element) {
    // 파라미터로 전달되는 element는 배열의 모든 원소가 순차적으로 전달된다.
    //console.log(element);

    //  element를 우리가 원하는 조건에 충족하는지 검사하여 true/false 리턴
    //  true를 리턴하는 순간 배열의 탐색을 중단. (break와 동일 기능)
    if (element%2==0){
        //  true가 리턴되는 경우 element는 found에 저장된다.
        return true;
    } else {
        //  false가 리턴되는 경우 element는 버려진다.
        return false;
    }

});
console.log(found);

출력

12

배열에서 특정 조건 충족하는 원소 추출

Array.filter(element => 조건식);

조건이 true일 경우에만 element들을 반환

arr = Array.filter(element => 조건식);
조건식이 true일 경우에만 Array의 element들이 arr에 들어간다.

Array.filter( function (element, index, arr) { 조건문에 따른 return}

ex)

code

const arr33 = [5,12,8,131,44];
const results = arr33.filter(function (element, index, arr) {
    console.log("element = %d, index = %d, arr = %s", element, index, arr);
    if (element % 2 ==0){
        //  true가 리턴되는 경우 element 는 results 배열의 원소로 저장된다.
        //  true를 리턴하더라도 배열의 모든 원소를 탐색하기 전까지는 종료되지 않는다.
        return true;
    } else {
        //false가 리턴되는 경우 element는 버려진다.
        return false;
    }
});
console.log("filter 함수 결과");
console.log(results);

출력

filter 함수 결과
[ 12, 8, 44 ]

배열 정렬

Array.sort()

퀵정렬 알고리즘을 사용하여 배열 자체를 정렬한다.

⇒ 배열의 모든 원소를 문자열로 취급하기 때문에 글자 정렬기준이 적용된다.

정렬조건을 콜백함수로 처리한다.

sort함수를 이용한 오름차순

sort함수를 이용한 내림차순

code

const Array2 = [2,1,15];
Array2.sort(function (a,b) {
	console.log("a=%s, b=%s", a,b);
	return b - a;
});
console.log(Array2);

출력

a=15, b=1
a=15, b=1
a=15, b=2
[ 15, 2, 1 ]
